# v0.3.1 Stable

这是 Skylattice 当前稳定版本的 canonical 页面。它把 Phase 5 的 operational closure 固定为可引用、可验证、可继续演进的稳定基线。

## 这个版本证明了什么

- weekly radar safe-validation 现在有两份 tracked 记录，验证闭环不再只靠一次孤立运行。
- OpenAI provider 的人类可读指令现在来自 `prompts/system/`，运行时代码只保留插值、校验、解析和 enforcement。
- 发布时的验证叙事更加诚实：无凭据基线继续保持公开可复现，认证相关检查则明确为显式、可选、可记录的 operator 步骤。

## 现在可以做什么

- 运行 `skylattice doctor`
- 运行带有 deterministic edit modes 和 GitHub collaboration recovery 的 `task-agent`
- 使用 `skylattice memory ...` 做 review、search、export、rollback
- 运行 `technology-radar`
- 使用 `skylattice radar schedule show`、`render`、`run`、`validate`
- 使用只读 auth preflight 和显式 GitHub bridge 检查 live adapter 准备情况

## 为什么这个稳定版本重要

`v0.3.0` 打开了 Phase 5 的调度与 provider-neutral foundation；`v0.3.1` 则把这条线收口为更清晰的 operator story：验证可以重复，prompt 真值源更明确，认证边界也更容易信任。

## 当前限制

- task 执行仍然是 text-native，而不是 AST-aware
- 有意义的 task planning 仍依赖 `OPENAI_API_KEY`
- radar discovery 仍依赖 `GITHUB_TOKEN`
- 调度集成目前仍是 Windows-first，并依赖 OS-level task registration
- 如果你需要成熟的 hosted automation 平台，项目仍然偏早期
