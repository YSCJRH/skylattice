# v0.4.1 Stable

这是 Skylattice 当前稳定版本的 canonical 页面。

`v0.4.1 Stable` 不是重新定义产品方向，而是把 `v0.4.0` 打下来的 hosted web control-plane foundation 收得更稳：部署边界更清楚、浏览器侧状态更诚实、维护者入口更完整。

## 这个版本证明了什么

- Preview、本地开发态、Hosted Alpha blocked、以及真实 paired control 现在区分得更明确
- production-style Hosted Alpha 部署不会再静默回落到 local-file control-plane state
- 仓库里现在有更完整的 Hosted Alpha 部署 contract，包括 env template、runbook、bootstrap SQL 和检查命令

## 现在可以做什么

- 运行 `skylattice doctor`
- 使用 governed `task-agent`
- 使用 `skylattice memory ...`
- 使用 `technology-radar`
- 使用 `skylattice web status`、`web pair`、`web connector ...`
- 在 `apps/web` 里查看 dashboard、tasks、radar、memory、commands、devices、approvals、connect
- 使用 `npm run web:hosted-alpha:check` 和 `npm run web:hosted-alpha:bootstrap` 做 Hosted Alpha 维护操作

## 为什么这个 patch 重要

`v0.4.0` 让 Skylattice 首次具备 hosted web control-plane 的产品外形；`v0.4.1` 则把这条线从“能看、能跑”继续推进到“更诚实、更可维护、更适合后续真实 Hosted Alpha 部署”。

## 当前限制

- 这个仓库目前仍然没有直接提供公开 hosted app URL
- 真实 Hosted Alpha 仍然需要在仓库外配置 Vercel、Neon 和 GitHub OAuth
- task 执行仍然是 text-native，而不是 AST-aware
- 有意义的 task planning 仍然依赖 `OPENAI_API_KEY`
