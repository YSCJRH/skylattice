# v0.4.0 Stable

这是 Skylattice 当前稳定版本的 canonical 页面。

`v0.4.0 Stable` 把项目从“文档站 + 本地 CLI/runtime”推进到了“有真实 hosted control plane 的产品形态”，但执行、记忆和治理仍然保持 local-first。

## 这个版本证明了什么

- Skylattice 现在已经有了真正的网页产品层
- browser 侧可以承载 pairing、command history、command drill-down、device 管理、approval reminders
- 这些新能力没有把 runtime truth 从本地迁移到云端

## 现在可以做什么

- 运行 `skylattice doctor`
- 使用 `task-agent`
- 使用 `skylattice memory ...`
- 使用 `technology-radar`
- 使用 `skylattice web status`、`web pair`、`web connector ...`
- 在 `apps/web` 里本地查看 dashboard、tasks、radar、memory、commands、devices、approvals 等页面

## 为什么这个稳定版本重要

`v0.3.1` 主要是收口 Phase 5 的 operator story。`v0.4.0` 则把产品外形真正往前推了一步：现在 Skylattice 已经有一个可构建、可配对、可查看命令和设备状态的 hosted web control plane。

## 当前限制

- 这个仓库目前还没有公开托管的 web app URL
- 现在最现实的初步入口仍然是本地：`npm run web:dev`
- task 执行仍然是 text-native，不是 AST-aware
- 有意义的 task planning 仍然依赖 `OPENAI_API_KEY`
