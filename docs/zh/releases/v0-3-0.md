# v0.3.0 Stable

这是 Skylattice 当前稳定版本的 canonical 页面。它把 Phase 4 的 GitHub 协作收口，以及 Phase 5 的本地调度基础，一起固定为可引用、可验证、可继续演进的稳定基线。

## 这个版本证明了什么

- task-agent 的远端协作尾部现在是可检查、可恢复、可审计的：推送、PR 预观察、PR 同步、issue comment 去重恢复都能在本地 inspect/recovery 中看清楚。
- radar 现在有 tracked 的 schedule config，并通过本地系统计划任务集成来运行，而不是引入仓库内常驻调度器。
- radar discovery 已经抽象出稳定的 source protocol，为后续多源扩展预留了结构，但当前仍只启用 GitHub provider。

## 现在可以做什么

- 运行 `skylattice doctor`
- 执行带 memory context 和恢复摘要的 `task-agent`
- 使用 `skylattice memory ...` 做 review、search、export、rollback
- 运行 `technology-radar`
- 使用 `skylattice radar schedule show`
- 使用 `skylattice radar schedule render --target windows-task`
- 使用 `skylattice radar schedule run`

## 为什么这个稳定版本重要

`v0.2.2` 让 Skylattice 的 authority kit 和稳定版本信号更完整，`v0.3.0` 则把稳定线推进到真正的运行时与运维边界：GitHub 协作恢复更加显式，radar 也拥有了第一版本地调度基础。

## 当前限制

- task 执行仍然是 text-native，而不是 AST-aware
- 有意义的 task planning 仍依赖 `OPENAI_API_KEY`
- radar discovery 仍依赖 `GITHUB_TOKEN`
- 调度集成目前按 Windows-first 设计，依赖本地操作系统计划任务
- 如果你需要成熟的托管式自动化平台，项目仍然偏早期
