# v0.2.1 Stable

This is the canonical public release page for the first non-pre-release Skylattice release.

## What This Release Proves

- Skylattice now has a conventional stable release signal that search engines, AI answer systems, and directory curators can cite.
- The English and Chinese answer-first public pages are aligned around the same core positioning: local-first memory, governed repo tasks, and Git-native reviewability.
- The repository now tracks both proof surfaces and outreach assets, so the public story is easier to verify and repeat externally.

## What Works Today

- `skylattice doctor`
- governed `task-agent` runs with deterministic edit modes
- `technology-radar` runs with GitHub discovery and bounded promotion metadata
- Windows-first CI aligned with the tracked validation config
- redacted public proof artifacts under `examples/redacted/`

## Why The Stable Signal Matters

A non-pre-release cut gives Skylattice a conventional `latest release` signal on GitHub. That makes the project easier to cite in search results, AI answer systems, directory submissions, launch posts, and comparison pages.

## Known Limits

- task execution remains text-native rather than AST-aware
- meaningful task planning still depends on `OPENAI_API_KEY`
- radar discovery still depends on `GITHUB_TOKEN`
- automatic promotion remains constrained to whitelisted tracked paths
- the project is still early if you need a broad hosted automation platform
