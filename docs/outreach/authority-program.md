# External Authority Program

Skylattice now has the technical packaging needed for citation. The next job is to earn a small number of legitimate external signals that help search engines and AI answer systems trust the official Pages surface.

## Program Goal

Over 30 days, publish and distribute a compact set of external-facing assets that increase three things:

- branded and non-brand mentions of Skylattice outside GitHub
- backlinks to the canonical Pages surfaces
- confidence that the project is real, maintained, and worth citing

## What Counts As Success

- one stable release page is consistently used as the canonical version anchor
- one English launch post is published
- one Chinese launch post is published
- one comparison/category post is published
- three to five legitimate directory, list, or community references point back to Skylattice
- Day 14 or Day 30 review shows some improvement in official-source citations or branded-query traffic

## Core Message To Repeat

Skylattice is a local-first AI agent runtime for persistent memory, governed repo tasks, and Git-native reviewability.

## 30-Day Sequence

### Days 1-3

- finalize the English launch post
- finalize the Chinese launch post
- finalize the category/comparison article
- choose the first three external targets from the distribution tracker

### Days 4-7

- publish the English launch post
- publish one short English community post
- submit at least one relevant directory or list entry

### Days 8-14

- publish the Chinese launch post
- publish one short Chinese community post
- submit two more relevant directory or roundup entries
- record the Day 14 review in `evals/ai-search/`

### Days 15-21

- publish the category/comparison post
- post one follow-up comparison-oriented community summary
- refresh backlinks so the strongest Pages pages keep being cited

### Days 22-30

- do one final wave of high-fit community posting
- record the Day 30 review in `evals/ai-search/`
- decide whether the next cycle should focus on more authority-building or new query-shaped pages

## Rules

- post only where Skylattice is a real fit
- always link back to a canonical Pages page, not a random GitHub blob URL
- avoid keyword stuffing or shallow ?project drop? comments
- prefer one good post with proof links over five vague launch messages
