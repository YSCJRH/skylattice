# Posting Runbook

Use this runbook when Skylattice is ready for external mentions and backlinks.

## Before Posting Anything

1. Confirm these URLs are live:
   - homepage
   - what-is page
   - quick start
   - proof page
   - `v0.3.0` stable release page
2. Confirm the GitHub repository header still matches the Pages positioning.
3. Confirm the latest release on GitHub is `v0.3.0 Stable`.
4. Confirm the social preview image is still configured on the repository.

## Launch Post Checklist

- choose one primary link target and one proof link
- mention the stable release explicitly
- mention the no-credential verification path explicitly
- mention why Skylattice is different from broader agent frameworks
- end with one clear CTA: star, read the proof page, or try the quick start

## Community Post Checklist

- shorten to one audience-specific message
- avoid pasting the full launch post
- keep one primary link and one optional secondary proof link
- answer likely follow-up questions before posting:
  - is it runnable today?
  - is it local-first?
  - is it a generic coding agent framework?

## Directory / List Submission Checklist

- use the short blurb from `directory-blurbs.md`
- choose the best-fit category, not the broadest one
- include the homepage and stable release page when possible
- if the directory supports tags, prefer:
  - local-first
  - ai-agent
  - agent-framework
  - memory
  - cli
  - sqlite

## After Each External Mention

- record the URL and date in `distribution-targets.md`
- note which canonical link target was used
- note whether the mention used the correct positioning
- schedule it for inclusion in the next discoverability review summary
