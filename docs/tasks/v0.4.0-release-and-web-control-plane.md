# Task Brief: v0.4.0 Release And Web Control Plane

## Intent

- publish a stable minor release that turns the new hosted web control-plane foundation into the latest stable public signal
- success means version metadata, release notes, Pages surfaces, public-readiness tests, and the GitHub release all point at the same `v0.4.0 Stable` story

## Constraints

- tracked artifacts to edit: version metadata, release docs, README and Pages references, AI-distribution surfaces, public-readiness tests, and this task brief
- local-only artifacts expected to change: optional release-body staging file under `.local/work/` for GitHub release publishing
- explicit non-goals: no hosted execution runtime, no deployment of a public multi-tenant app, no governance widening, and no removal of the current CLI-first local runtime

## Affected Subsystems

- package and citation metadata
- public Pages and release surfaces
- hosted web control-plane positioning
- AI-distribution and outreach references
- public-readiness verification and release publishing

## Verification

- `npm run web:lint`
- `npm run web:build`
- `python -m pytest -q`
- `python -m compileall src/skylattice`
- `python -m skylattice.cli doctor`
- `python -m skylattice.cli web status`
- `python -m mkdocs build --strict`
- `git diff --check`
- manual checks for `gh release view`, Pages/latest-release references, and current `main` state

## Notes

- default release assumption: `v0.4.0 Stable` as a minor follow-up to `v0.3.1 Stable`
- release body should describe the hosted web control-plane foundation, local bridge API, local connector, and productization gaps that still remain
- `v0.3.1 Stable` remains a historical stable milestone and should stay linked in the release history
